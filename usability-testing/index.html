UX Action Plan & Iterative Recommendations

High Severity

Stock Watchlist Navigation Fold

Participants took over 14s to discover Apple stock because watchlist elements sat below the primary screen viewport fold, violating Heuristic 7 (Efficiency of Use).

Proposed Re-design Fix: Integrate a global search icon directly in the main header of the dashboard, allowing users to type equity searches directly without vertical page scrolls.
Medium Severity

Onboarding Double-Tap Slide Jumps

Users frequently double-clicked 'Next' on slides due to feedback delays, causing them to jump straight to authentication and skip tutorials (Heuristic 1: Status Visibility).

Proposed Re-design Fix: Introduce a brief transition timeout (300ms) that disables next-navigation clicks during card slide animations, preventing duplicate commands.
Low Severity

Onboarding Indicator Dots Dot Targeting

Some users tried to tap the small indicator slider dots to jump slides directly, but targets were not wired with click handlers (Heuristic 7: Flexibility).

Proposed Re-design Fix: Wire click event listeners on dot coordinates, converting them to active navigational shortcuts that skip slides directly.

The Scoring Algorithm

For odd questions, subtract 1 from the raw response (0-4 scope). For even questions, subtract the response value from 5 (0-4 scope). Sum these values and multiply the result by 2.5 to scale the final output to 100.

Score = [ Sum(Qi_odd - 1) + Sum(5 - Qi_even) ] × 2.5

Usability Research Test Plan

Official testing parameters, objectives, scripts, and criteria used to run participant sessions for the Zenith App.

1. Executive Objectives

  • Evaluate Learnability: Measure time spent decoding dashboard columns and navigating stock watchlists.
  • Identify Navigation Friction: Count errors when clicking navigation items and searching stock details.
  • Benchmark Transactions: Track order checkout completion rates and input error rates.
  • Quantify Usability: Gather standardized SUS survey metrics to measure overall user satisfaction.

2. Participant Profile Recruitment

We recruited 8 participants based on the following criteria:

  • Ages 22-65, balanced gender splits.
  • Experience ranges: 4 novice investors (never traded or used apps), 4 expert traders (use mobile investment brokerages daily).
  • In-person sessions (5) conducted in our Zenith Design Lab; Remote sessions (3) observed via screen sharing.

3. Testing Scenario Scripts

Scenario 1: System Boot & Carousel Onboard

Script: "You are booting the Zenith Wealth application for the first time. Spend a moment reviewing onboarding tutorial cards, then advance to the authentication screen."

Scenario 2: Secure Log In

Script: "Log in securely to view your asset records. Enter any code inside the account security pin field, click submit, and confirm you see the dashboard."

Scenario 3: Stock Asset Discovery

Script: "You wish to purchase stock in Apple Inc. Navigate to the stock detail page for AAPL and confirm you can view price statistics and charts."

Scenario 4: Order Settle & Settle

Script: "Execute a purchase order for $100 of Apple Stock. Verify transaction summary prompts, finalize order checkout, and return safely to dashboard."

4. Key Usability Metrics Tracked
